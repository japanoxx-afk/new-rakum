# 라크무 반응지연 동적분석 가이드 (명령 지연 상수 찾기)

목표: 싱글(AI 대결) 게임 중 **턴 카운터**와 **명령 지연 턴 수(N)** 를 직접 관찰해
정확한 코드 위치를 특정 → 지연을 줄이는 패치를 만든다.

도구: **Cheat Engine** (무료). 이 작업의 핵심 기능 "이 주소에 접근하는 코드 찾기"가
디버거보다 훨씬 쉽고 정확하다. https://cheatengine.org 에서 설치.

> 라크무는 32비트라 CE 기본 설정으로 그대로 동작한다.

---

## 1단계: 턴 카운터 찾기 (게임 속도의 심장)

1. 라크무 실행 → **싱글/AI 대결로 인게임 진입** (유닛이 보이는 상태).
2. Cheat Engine 실행 → 좌상단 컴퓨터 아이콘 → 프로세스 목록에서 **Rhakmu.exe** 선택.
3. `Value Type` = **4 Bytes**, `Scan Type` = **Unknown initial value** → **First Scan**.
4. 게임을 그대로 둔 채(유닛 가만히), 약 1초 간격으로
   `Scan Type` = **Increased value** → **Next Scan** 을 5~8회 반복.
   - 계속 증가하는 값만 남는다(턴 카운터·게임시계·프레임수 등 후보).
5. **턴 카운터만 골라내기**: 게임을 일시정지(메뉴 ESC 등)했다가
   - 정지 상태에서 `Unchanged value` → Next Scan
   - 재개 후 `Increased value` → Next Scan
   이렇게 하면 "시뮬레이션 턴에 묶인 값"만 남는다. (실시간 ms 시계는 정지 중에도 변해서 걸러진다)
6. 남은 후보(보통 1~5개)를 더블클릭해 아래 테이블에 추가.
   - 각 값이 **초당 몇 번 증가하는지** 눈으로 세어 메모 → 이게 **턴 레이트(turns/sec)**.
     예: 초당 6~10회면 턴 간격 ≈ 100~166ms.

> **📌 보고 1**: 턴 카운터 후보 주소들과 각각의 초당 증가 횟수.

---

## 2단계: 턴 카운터를 쓰는 "명령 스케줄러" 잡기

1. 진짜 턴 카운터로 보이는 후보를 우클릭 → **Find out what accesses this address**.
2. 작은 창이 뜨면, 게임으로 돌아가 **유닛 이동 명령**을 내린다
   (유닛 클릭 → 땅 우클릭). 공격·건설 명령도 몇 번 섞어 준다.
3. 접근 명령어 목록이 쌓인다. 그중 **매 클릭마다 한 번씩** 실행되는 항목이
   명령 스케줄러다(매 프레임 도는 렌더/AI 접근과 구분됨).
4. 그 항목을 더블클릭 → 디스어셈블이 보인다. 주변에서 다음을 찾는다:
   - `mov eax, [턴카운터]` 직후 **`add eax, N`** 또는 **`lea eax, [eax+N]`**
     → 이 **N이 명령 지연 턴 수**(우리가 줄일 값).
   - 그 명령어의 **주소**(예: `0x004xxxxx`)도 기록.

> **📌 보고 2**: 클릭마다 한 번 접근하는 명령어의 주소 + 주변 5~10줄 디스어셈블
> (특히 `add/lea ..., N` 부분). 스크린샷이나 텍스트 복붙 모두 OK.

---

## 3단계: 턴 증가(쓰기) 지점도 확인 (선택)

1. 턴 카운터 우클릭 → **Find out what writes this address**.
2. 게임을 잠깐 진행하면 **한 군데**가 잡힌다 = 턴 전진 코드.
3. 그 주소 기록.

> **📌 보고 3**: 턴 카운터에 쓰는 명령어 주소 1개.

---

## 내가 할 일 (보고 받은 뒤)

- 보고 1·2로 **턴 간격 × 지연 턴 수 = 실제 지연(0.3~0.5s)** 을 검증.
- 명령 지연 N을 줄이는 **in-place 패치**를 만들어 `Apply-RhakMuStable.ps1`에 추가.
  - 예: `add eax, 3` → `add eax, 1` (지연 3턴 → 1턴)처럼 안전 범위로 축소.
- 싱글에서 즉시 체감 테스트 → 안정되면 멀티에도 동일 적용
  (멀티는 양쪽 PC 동일 패치 + 네트워크 RTT가 줄인 창 안에 들어와야 함).

## 참고: 빠른 지연 측정만 먼저 하고 싶다면
1단계까지만 해서 **턴 레이트(turns/sec)** 만 알려줘도, 지연의 절반(턴 간격)이 특정된다.
거기에 체감 지연(0.3~0.5s)을 나누면 대략의 지연 턴 수도 역산할 수 있다.
