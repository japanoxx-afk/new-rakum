# 라크무 밸런스 수정 가이드

유닛/건물의 체력·공격력·방어력·가격·빌드타임 등을 Excel로 편집할 수 있습니다.
게임 데이터(`Data001.FDB`) 안의 STB 표를 CSV로 꺼내 수정하고 다시 집어넣는 방식입니다.

## 준비물
- Python 3 (이미 설치되어 있음: `C:\Users\seo\AppData\Local\Programs\Python\Python314\python.exe`)
- 도구: `stb_tool.py`

## 4단계 작업 흐름

```powershell
$py = "C:\Users\seo\AppData\Local\Programs\Python\Python314\python.exe"
cd "C:\Users\seo\Documents\라크무-claude"

# 1) 게임에서 STB 표 추출 (최초 1회)
& $py stb_tool.py extract

# 2) 수정할 표를 CSV로 내보내기 (Excel에서 열기)
& $py stb_tool.py export R1_DATA_CHAR.STB     # 유닛
& $py stb_tool.py export R1_DATA_CNST.STB     # 건물

# 3) extracted_stb\R1_DATA_CHAR.csv 를 Excel로 열어 값 수정 후 저장
#    (반드시 CSV UTF-8 형식으로 저장)

# 4) CSV -> STB 변환 후 게임에 적용
& $py stb_tool.py import R1_DATA_CHAR.STB
& $py stb_tool.py apply  R1_DATA_CHAR.STB     # Data001.FDB 에 주입 (자동 백업)
```

적용 후 게임을 재시작하면 변경값이 반영됩니다.

## 주요 표 / 컬럼

| 파일 | 내용 | 핵심 컬럼 |
|------|------|-----------|
| `R1_DATA_CHAR.STB` | 유닛 | `캐릭이름`, `수명`(체력), `공격력좌/우`(공격력), `소내성/마내성`(방어) |
| `R1_DATA_CNST.STB` | 건물 | `건물이름`, `수명`(체력), 건설비용·빌드타임 등 |
| `R1_DATA_SKILL.STB` | 스킬 | 스킬 데이터 |
| `R1_DATA_UPGL.STB` | 업그레이드 | 업그레이드 데이터 |
| `R1_DATA_DAMAGE.STB` | 데미지 배율표 | 공격타입 vs 방어타입 |

각 표의 **첫 번째 데이터 행(행 0)은 설명행**이므로 건드리지 마세요. 실제 데이터는 그 아래부터입니다.
(예: CNST는 `위국성` 체력 3000, `초가` 800 부터가 실제 건물)

## 주의사항
- **숫자 길이 제한**: 수정값을 원래보다 너무 길게(자릿수↑) 늘리면 `apply` 단계에서
  "modified file larger" 오류가 날 수 있습니다. 그럴 땐 다른 빈 값을 줄여 전체 크기를 맞추세요.
  (보통 3000→5000, 800→400 처럼 같은 자릿수 변경은 항상 안전)
- **백업**: `apply` 첫 실행 시 `Data001.FDB.bak` 자동 생성. 원복하려면 이 파일을 복사해 덮어쓰면 됩니다.
- CSV는 **UTF-8 (BOM)** 인코딩이어야 한글이 깨지지 않습니다 (도구가 그렇게 저장함).

## 멀티플레이 주의
밸런스 수정은 **모든 플레이어가 동일한 `Data001.FDB`를 사용**해야 합니다.
한쪽만 수정하면 게임 동기화(lockstep)가 깨져 desync/튕김이 발생합니다.
수정한 `Data001.FDB`를 함께 플레이할 모든 PC에 배포하세요.
